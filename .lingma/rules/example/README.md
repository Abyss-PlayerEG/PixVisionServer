# 代码注释示例文件说明

本目录包含项目代码注释和 API 文档的标准示例，用于指导开发者编写规范的注释。

---

## 📁 文件列表

### 1. JavadocCommentExample.java
**注解类 Javadoc 注释示例**

展示了如何为自定义注解编写完整的 Javadoc 注释，包括：
- 类级别的详细说明（使用场景、示例、注意事项、最佳实践）
- 方法/属性级别的参数说明
- 标准 HTML 标签格式化（`<h3>`、`<pre>`、`<ul>`、`<ol>`）
- `@author`、`@see`、`@since` 等标准标签

**适用场景**：自定义注解、工具类、复杂业务类

---

### 2. ServiceCommentExample.java
**Service 层注释示例**

展示了如何为 Service 接口和方法编写完整的 Javadoc 注释，包括：
- 接口的功能说明和使用场景
- 方法的详细文档（参数、返回值、异常）
- 完整的使用示例代码块
- 业务规则和约束条件说明

**适用场景**：Service 接口、Service 实现类

---

### 3. ControllerCommentExample.java
**Controller 层注释示例**

展示了 Controller 层的简洁注释风格，包括：
- Controller 类的简要功能说明
- 方法的简洁 Javadoc（仅包含简要描述、参数、返回值、作者）
- Swagger `@Operation` 注解的完整结构
- 详细的 API 文档在 Swagger 中提供，避免重复

**适用场景**：所有 Controller 类

---

### 4. SwaggerDocExample.java
**Swagger API 文档示例**

展示了如何编写规范的 Swagger API 文档，包括：
- `@Tag` 注解的使用
- `@Operation` 注解的完整结构（特性、参数说明、返回说明、业务逻辑、注意事项）
- `@Parameter` 注解的参数说明
- 一级标题格式规范：`功能名称（需要|无需登录认证 + 角色权限[XX]）`
- **禁止包含 JSON 数据结构示例**

**适用场景**：所有需要生成 API 文档的 Controller 接口

---

## 📝 核心规范要点

### Javadoc 注释规范

**必须包含的要素：**
1. **简要描述**：第一行简短说明功能
2. **详细说明**：使用 `<p>` 标签分段说明
3. **使用场景**：`<h3>使用场景</h3>` + 有序列表
4. **使用示例**：`<h3>使用示例</h3>` + `<pre>{@code}</pre>` 代码块
5. **注意事项**：`<h3>注意事项</h3>` + 无序列表
6. **最佳实践**：`<h3>最佳实践</h3>` + 无序列表（可选）
7. **作者信息**：`@author`
8. **相关引用**：`@see`
9. **版本信息**：`@since`（可选）

**Controller 层特殊规范：**
- Controller 已有 Swagger 文档，Javadoc 应保持**简洁**
- 只需包含：简要描述、参数说明（`@param`）、返回值说明（`@return`）、作者信息（`@author`）
- **不需要**：使用场景、使用示例、注意事项、最佳实践等详细内容

### Swagger 文档规范

**必须包含的章节：**
1. **一级标题**：`# 功能名称（需要|无需登录认证 + 角色权限[XX]）`
2. **特性**：接口的核心功能和特点
3. **参数说明**：参数的类型、必填性、取值范围
4. **返回说明**：文字描述返回内容（**禁止 JSON 结构**）
5. **业务逻辑**：步骤化的处理流程
6. **注意事项**：重要的使用限制和警告

**禁止事项：**
- ❌ **禁止在注释中使用 emoji 表情符号**（如 ✅ ❌ 💡 ⚠️ 等）
- ❌ **禁止在 Swagger 文档中包含 JSON 数据结构示例**
- ❌ **禁止在 Swagger 文档中包含 curl/HTTP 请求示例**
- ❌ 禁止省略关键参数的说明
